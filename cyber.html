Effectuer un Pentesting

J'ai mené un pentesting sur la machine Phineas 1, commençant par la découverte de l'IP cible avec netdiscover. Après avoir identifié l'adresse IP suspecte, j'ai utilisé nmap pour confirmer la présence d'un serveur web Apache 2.4.6 et détecté une vulnérabilité potentielle avec la méthode TRACE. Ensuite, j'ai exploré le site via un navigateur et confirmé qu'il s'agissait d'une page par défaut d'Apache. J'ai utilisé Nikto pour analyser plus avant, découvrant une possible vulnérabilité liée à l'outil curl, et j'ai procédé à une énumération avec Gobuster pour trouver des répertoires et fichiers cachés, ce qui m'a conduit à découvrir le CMS Fuel.


Après avoir identifié Fuel comme un CMS, j'ai trouvé la page de login en tentant différentes URL. J'ai ensuite cherché des exploits connus pour Fuel CMS sur ExploitDB, trouvant un script Python pour une attaque de type RCE. L'exploitation de cette vulnérabilité m'a permis de m'introduire dans le système en tant que systemapache. J'ai ensuite trouvé les informations de connexion de l'utilisateur 'anna' dans le fichier database.php, ce qui m'a permis de me connecter via SSH. En explorant le système, j'ai découvert une vulnérabilité d'injection via pickle dans un fichier 'app.py', que j'ai exploité pour obtenir des privilèges root, trouvant ainsi le fichier root.txt.
